3.阐述对Docker了解

​ docker是一个开源的应用容器引擎，可以将打包的应用移植到镜像中。然后发布到运行的Linux或windows机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

​ docker基于go语言并遵从apache2.0协议开源

阐述仓库 容器 镜像之间关系

可以在仓库中拉取镜像，容器是由镜像创建，容器和镜像的关系类似面向对象中类与对象，镜像能够创建多个容器。

Docker的优势和应用场景是什么

  • docker的优势：快速部署，高扩展性

  • docker的应用场景：以集装箱的形式快速部署企业级的应用系统

    ​ 高扩展、分布式、集群和微服务

    ​ 大数据 云计算…

4.请说出关系型数据库的特征 常用的产品
  • 关系型数据库的特征：X Y轴构建的二维表、有行列的区别、满足三大范式、通过加密后的文件形式存放于磁盘、有事务的ACID特性、支持通用SQL
  • 常用的产品：MySQL、Oracle、IBM–DB2、informix

说出对视图的了解

将一段查询sql语句封装为虚拟表 ；这个虚拟表只保存sq逻辑 不会保存任何查询结果。

封装复杂sql语句，提高复用性；逻辑放在数据库上面，更新不需要发布程序，面对频繁的需求变更 更灵活。

create view view_name as 查询语句

drop view view_name

索引的了解

索引（Index）是帮助MySQL高效获取数据的数据结构。

create index index_name on mytable(no,name)

5.请说出非关系型数据库的特征 常用的产品
  • 非关系数据库特征：键值对的形式、数据类型丰富、一款内存数据库、提供内存中数据进行持久化
  • 常用产品：Redis

Redis数据库的特征

  • 易于大型企业系统快速扩展：具有主从复制 哨兵机制
  • 高性能的读写速度
  • 使用C语言开发 更加有利于硬件资源的管理
  • 分布式集群
  • 弥补关系型数据的不足、高缓存、消息订阅、关系维护
  • 具有16个逻辑数据库

阐述对MyCat的了解

  • 数据库中间件：处于底层数据库和用户应用系统之间，主要用于屏蔽异构数据库的底层细节的。

    mycat是一个彻底开源，面向企业应用开发的大数据集群。

    基于心跳的自动故障切换、分库分表、读写分离

Spring MVC入门

  1. HTTP超文本传输协议：用于传输HTML等内容的应用层协议

    规定了浏览器和服务器之间如何通信，以及通信时的数据格式。

  2. 三层架构：表现层、业务层、数据访问层

    MVC：模型层Model、View视图层 、 Controller控制层

    核心组件：前端控制器 - DispatcherServlet

  3. Thymeleaf

    模板引擎：生成动态的HTML

    ​ 倡导自然模板，即以HTML文件为模板

    ​ 标准表达式、判断与循环、模板的布局

  4. responsebody:返回JSON可以添加 返回html页面可以不用添加

MyBatis入门
