gcc编译过程

#include< > 与 #include “”的区别：

  • < > 表示系统直接按系统指定的目录检索
  • “”表示系统先在 “” 指定的路径(没写路径代表当前路径)查找头文件，如果找不到，再按系统指定的目录检索

64位和32位系统区别

寄存器是CPU内部最基本的存储单元。

CPU通过总线（地址、控制、数据）来和外部设备交互，所有的64位CPU兼容32位的指令，32位要兼容16位的指令，所以在64位的CPU上是可以识别32位的指令。

在64位的CPU架构上运行了64位的软件操作系统，那么这个系统是64位。

在64位的CPU架构上，运行了32位的软件操作系统，那么这个操作系统就是32位

64位的软件不能运行在32位的CPU上

有符号数和无符号数区别

二进制下有符号数最高位为符号位，0代表正数，1代表负数

无符号数最高位不是符号位，而就是数的一部分，无符号数不可能是负数。

sizeof关键字

  • sizeof不是函数，所以不需要包含任何头文件，作用是计算一个数据类型的大小，单位为字节
  • sizeof返回值为size_t
  • size_t类型在32位操作系统下是unsigned int，是一个无符号的整数
  • 根据不同系统，使用printf()函数显示sizeof表达式，可能使用%u、%lu、%llu.

CPU计算时，预先把要用的数据从硬盘督导内存，然后再把即将用的数据读到寄存器。

缓存：因为如果经常操作内存中的同一地址的数据，就会影响速度。于是就在寄存器与内存之间设置一个缓存。

C99标准：ANSI/ISO联合委员会（C9X委员会）开始修订C标准，最终发 布了C99标准。该委员会遵循了最初C90标准的原则，包括保持语言的精练 简单。

C11：标准委员会在2007年承诺C标准的下一个版本是C1X，2011年终于发布了C11标准。

转义字符

_Bool类型

C99标准添加了_Bool类型，用于表示布尔值，即true和flase。因为C语言中用值1表示true，值0表示false，所以 _Bool类型实际上也是一种整数类型。但原则上它只占用1位（bit）存储空间。

字符串

C语言中没有专门用于存储字符串的变量类型。字符串都被存储在char类型的数组中。数组有利阿努的存储单元组成，字符串中的字符被储存在相邻的存储单元中，每个单元储存一个字符。

数组的末尾的字符时’\0’。这是空字符，用空字符标记字符串的结束。它时非打印字符，其ASCII码值（或等价于）0.

‘\0’ == 0, ‘0’ != ‘\0’

字符与字符串常量的区别

字符串常量**”x”和字符常量‘x’**不同：

  1. **’x’是基本类型(char)，而“x”**是派生类型(char数组)
  2. “x”实际是由两个字符组成：‘x’和空字符‘\0’

自动类型转换

  1. 设计两种类型的运算，两个值会被分别转成两种类型的更高级别。（类型的级别从高到低一次是long double、double、float、unsigned long long、long long、unsigned long、long、unsigned int、int)
  2. 再赋值表达式语句中，计算的最终结果会被转换成被赋值变量的类型。这个过程可能导致类型升级或降级。所谓降级，是指把一种类型转换成更低级别的类型。类型降级可能会丢失精度。

数组

格式：类型 数组名[元素个数] = {逗号分隔的初始化列表}

eg: int code[12] = {1,2,3,4,5};

常见初始化方式:

  • 对于一般初始化，在初始化一个元素后，未初始化的元素都会被设置为0

    int arr[200] = {0,100,200}

  • 省略初始化列表，元素的值为随机值

    char arr[1024]

  • 省略数组大小，元素大小为池化列表中值的个数

    double arr[] = {1,2,1.26,8.56};

  • C99规定，可以在初始化列表中使用带方括号的下标指明初始化的元素，后面这些值将被用于吃实话指定元素后面的元素。

    double aarr[5] = { [3] =1.2,1.8}

二维数组和多维数组

arr是一个拥有两个元素的一维数组，每个元素的类型是3个int类型数据的以为数组。

无论是几维数组，都应看作，数组元素为n-1维数组的一维数组。

应用：录入一个班5个人的三门成绩（语文、数学、英语）。然后计算每个同学成绩的总分，打印出来。

数组名是数组首元素的地址。

指针

指针变量的声明和定义：

 C语言中没有指针类型，指针实际上是一个符合类型，声明指针变量时必须指定指针所指向变量的类型，因为不同变量类型占用不同的存储空间，一些指针操作要求知道操作对象的大小。

类型说明符表明了指针所指向对象的类型，*表明声明的变量是一个指针。

指针的一些操纵

赋值：可以将地址赋给指针

解引用：解引用指向某个变量的指针，并且修改指向变量空间中的值

解引用相对数组名偏移2的指针，此时指向数组中的第3个元素

取址：

和所有变量一样，指针变量也有自己的地址和值。对指针而言，&运算符给出指针本身的地址。

指针与整数的加减：

指针和整数相加，结果是指针增加了整数个指针指向类型的大小。然后把结果与初始地址相加。

指针步长

指针加1指的是增加一个存储类型的大小。对数组而言，这意味着把加1后的地址是下一个元素的地址，而不是下一个字节的地址。

野指针

段错误原因：使用了未申请的空间。基本上都是由指针越界造成的。

const关键字

指向const的指针

因为const距离最近。所以const修饰的是 * ，表示指针指向的空间，被const修饰后，就不能更改。

const指针

const没有修饰指针指向的空间，修饰的是指针变量本身，因此，可以修改指针指向的空间，但是不能修改指针变量本身的值，也就是说，不能修改指针的指向。

指向const的const指针

const作用

地址传递是可以提高效率的，但是有时候想要效率。又不想让被调函数修改主调函数的变量，可以使用const限定指针。

如果函数的意图不是修改数组中的数据内容，那么在函数原型和函数定义中声明形式参数是应使用关键字const。
