C到C++基础过渡

1.C++对C语言加强

1.1 类型增强

1.1.1 const关键字

​ C语言中的Const修饰的变量，依然可以使用指向这块空间的非Const指针去改变其值。

​ C++中的Const为真正的Const常量，而不是C中的只读变量。在const修饰的常量编译期间，就已经确定下来了。C++中const修饰的只读变量空间，不能普通指针指向(也不能用解引用的方式间接修改这块空间的内容)，必须const修饰指针。

1.1.2 类型检测增强

类型必须一致才能相互赋值（除了本身可以进行隐式转换的类型之间）

C语言中重复定义多个相同名字的全局变量是合法的，但是C++不允许。

1.1.3 枚举增强

C++枚举为真正的枚举，C语言中的枚举变量其实就是interesting，可以任意赋值，而C++的枚举变量只能赋枚举值。

1.1.4 三目运算符

C语言中表达式不能作为左值，而C++某些表达式可以

1.1.5 struct增强

C语言的struct定义了一组变量的集合，C编译器并不认为这是一个新类型的定义声明

1.1.6 新增bool类型

C里面要想使用bool类型的变量，需要加上#include <stdbool.h>头文件，但是C++里面是可以直接定义bool型变量

1.3 new/delete

（1）对比C和C++

​ C: malloc()/free() 是函数

​ C++: new/delete 是关键字

（2）注意:

​ 配对使用，避免内存泄漏和多重释放。

​ 避免，交叉使用。比如 malloc 申请的空间去 delete，new 出的空间被 free;

​ 引入这两个关键字。此两关键字，重点用在类对象的申请与释放。

2.c++对C语言的扩展

2.1 引用

（1）变量名

​ 变量名本质上就是一段连续内存空间的名字，相当于一个标记。可以用变量名来申请并命名内存空间，可以借助变量名来使用这个空间。

（2）引用的概念

​ 变量名本身为一块内存的引用。引用其实就是给一个已有的变量再取一个别名（alias）。

（3）语法格式

​ 类型 &引用名 = 变量名;

（4）规则

​ 1.引用没有定义，只是一种关系类型声明，所以不分配内存。声明他和原有某一变量的关系，所以类型要保持一致。

​ 2.声明时必须初始化，一经声明,不可更改。

​ 3.可对引用再次进行引用。

​ 4.只有&前有类型时，是引用。若没有类型，则是其他意思。

（5）引用作为函数参数

普通引用在声明的时候必须用其他变量进行初始化，但是作为函数形参的时候，不需要进行初始化（因为这个初始化过程发生在函数调用的时候），实参给形参做初始化。

（6）引用作为函数返回

引用作为函数返回的时候，可以将这个函数作为赋值运算的左值（结果为赋值运算的值）

需要保证函数返回的引用空间在函数退出的时候没有被释放（全局变量、堆空间数据、传入指针的数据）

（7）引用的意义

①在很多时候引用可以代替指针使用

②引用在函数传参的使用中相对于指针具有更好的可读性和实用性

1）引用在C++中的内部实现是一个常指针： Type& name <===> Type* const name

2）C++编译器在编译过程中使用常指针作为引用的内部实现，因此引用所占用的空间大小与指针相同。

3）从使用的角度，引用会让人误会其只是一个别名，没有自己的存储空间。这是C++为了实用性而做出的细节隐藏。

函数传参好比间接赋值的3个必要条件：

1.定义两个变量（一个实参一个形参）

2.建立关联实参取地址传给形参

3.*p形参去间接的修改实参的值

引用在实现上，只不过是把：间接赋值成立的三个条件的后两步合二为一。当实参传给形参应用的时候，只不过是c++编译器帮我们手工取了一个实参地址，传给了形参引用（常量指针）

（9）const引用

const引用的特征

​ const引用有交付哦使用。它可以防止对象的值被随意修改。因而具有一些特性。

(1)const对象的引用必须是const的，将普通引用绑定到const对象是不合法的。既然对象是const的，表示它不能被修改，引用当然不能被修改，必须使用const引用。

(2)const引用可使用相关类型的对象(常量，得用类型的变量或表达式)初始化。这个是const引用与普通引用最大的区别。

const引用的原理

const引用的目的是，禁止通过修改引用值来改变被引用的对象。const引用的初始化特性较为微妙。
